# P1.1 Weekly Templates — Manual QA Checklist

Covers PR-E (data lift), PR-F (diff engine + API), PR-G (editor UI + history view).

Prereqs: logged in, at least one active phase with a few planned sessions in the next 14 days, at least one `calendar_events` row with a `day_code` bound to the active phase.

## /you — Weekly template section

- [ ] Section appears between the phase card and the exercises section.
- [ ] Header shows phase code badge and "Your training week".
- [ ] Summary line lists counts by type with rest last, e.g. `4 gym · 2 run · 1 rest`.
- [ ] Seven day tiles render Mon → Sun with correct type tinting.
- [ ] Today's tile shows the accent ring.
- [ ] Empty state renders cleanly when no active phase exists.
- [ ] "Edit →" link navigates to `/you/template?phase=<id>`.

## /you/template — Editor

- [ ] Page opens pre-populated with the current pattern (no flicker).
- [ ] Unauthed user is redirected to `/login`.
- [ ] Unknown `?phase=` redirects to `/you`.
- [ ] Phase selector lists all non-archived phases; switching phases reloads pattern.
- [ ] Per-day segmented control cycles through rest / gym / run / bike / swim / yoga / climb with icons.
- [ ] `day_code` dropdown disabled when type is `rest`; shows all `calendar_events.day_code` for the phase.
- [ ] Sticky footer toggles "Up to date" ↔ "Unsaved changes" as pattern mutates.
- [ ] "Cancel" returns to `/you` and discards edits.
- [ ] "Review changes" is disabled when pattern is unchanged (no pointless round-trip).

## Review modal

- [ ] Opens as bottom-sheet on mobile, centered card on desktop.
- [ ] Summary chips reflect server-computed diff (added / changed / removed / ai-preserved).
- [ ] Rationale text renders and is readable.
- [ ] Window scope shows `start → end` dates.
- [ ] Orphan warning block appears (warn tint) when `summary.orphan_day_codes` is non-empty.
- [ ] Details sections open automatically when the list has ≤ 6 items.
- [ ] "Confirm & apply" disabled while the preview is in-flight.
- [ ] Close/cancel returns to the editor with pattern intact.

## Apply edge cases

- [ ] No-op apply (pattern unchanged after round-trip) short-circuits without creating a history entry.
- [ ] 409 `version_conflict` shows toast and triggers `router.refresh()` so the editor reloads the latest server version.
- [ ] Success toast includes a "View" action linking to `/ai/<proposal_id>`.
- [ ] A logged session inside the window is preserved (counted under `skipped.updates_drifted`).
- [ ] An AI-proposed session in the window survives (counted under `summary.skipped_ai_proposed`).
- [ ] A past-planned delete target is skipped (counted under `skipped.deletes_not_planned`).
- [ ] A brand-new slot on a day that previously had none creates plans forward.

## /ai/[id] — template_change view

- [ ] Page routes through `TemplateChangeView` (not the adjust renderer) when `kind='template_change'`.
- [ ] "🗓 Template edit" chip and "Weekly shape updated" (or "…saved" when zero diff) headline render.
- [ ] Applied timestamp + `v<template_version>` appears in the sub-header.
- [ ] PatternCompare highlights changed rows with accent tint; unchanged rows muted.
- [ ] Stats grid pulls from `applied_counts` and matches the apply toast.
- [ ] Drift explanation paragraph appears only when `totalSkipped > 0`.
- [ ] Collapsible Added / Changed / Removed blocks render dates as `M/D` and slot text correctly (`type · day_code` or `Rest`).
- [ ] Scope footer matches the window the edit covered.
- [ ] Legacy adjust proposals still render normally (regression check).

## Data / RLS regression

- [ ] Edits scoped to the authed user only — opening `/ai/<other-user-proposal>` 404s.
- [ ] `weekly_templates` row created with correct `phase_id` + `version=1` on first save.
- [ ] Subsequent saves bump `version` monotonically.
- [ ] `ai_proposals` history row has `kind='template_change'`, `triggered_by='user_template_edit'`, `status='applied'`, `applied_at` set.
- [ ] Plans updated carry `source='template'` forward (sources for previously manual/AI rows preserved per engine rules).
